# Mapping Daily-Used Apps to Possible Attack Surfaces

In daily life, applications such as **Email, WhatsApp, and Banking apps** expose multiple **attack surfaces**—points where attackers may try to exploit weaknesses.

---

##1. Email Applications (Gmail, Outlook, Yahoo)

### Common Attack Surfaces
- Email attachments  
- Embedded links (URLs)  
- Login pages  
- Email servers  
- Password reset mechanisms  

### Possible Attacks
- Phishing attacks  
- Malware via attachments  
- Credential harvesting  
- Account takeover  

**Real-life example:**  
Clicking a fake “Reset your password” email link that redirects to a malicious website and steals login credentials.

---

## 2. WhatsApp / Messaging Applications

### Common Attack Surfaces
- Message links  
- Media files (images, videos, PDFs)  
- Account registration and OTP verification  
- Cloud backups  
- App permissions  

### Possible Attacks
- Phishing links in messages  
- Malware disguised as media files  
- SIM swapping (OTP interception)  
- Account hijacking  

**Real-life example:**  
A message stating *“Your WhatsApp account will be blocked, click here”* leading to a fake login page.

---

## 3. Banking Applications

### Common Attack Surfaces
- Login and authentication pages  
- OTP and SMS verification  
- Backend APIs  
- Mobile device local storage  
- Network communication (Wi-Fi, mobile data)  

### Possible Attacks
- Credential stuffing  
- Man-in-the-Middle (MITM) attacks  
- Malware stealing OTPs  
- Fake or malicious banking apps  

**Real-life example:**  
Accessing a banking app over unsecured public Wi-Fi, allowing attackers to intercept sensitive data.
